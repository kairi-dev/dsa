# DSA学習リポジトリ - Claude向けガイド

## プロジェクトの目的

AtCoderのような競技プログラミングではなく、**実務でDSAを使えるようにすること**が目的。
具体的には以下の領域での問題解決に使えることを目指す：
- 最適化（経路、リソース配分、パラメータチューニング）
- データ処理（センサーデータ、時系列、ストリーム）
- 予測（状態遷移、シーケンスモデリング）
- 制御（スケジューリング、イベント駆動）
- シミュレーション（離散事象、物理、モンテカルロ）

## 学習フェーズ

| Phase | トピック |
|-------|---------|
| 1: 基盤 | 01 計算量解析, 02 配列・NumPy |
| 2: 線形構造 | 03 連結リスト, 04 スタック・キュー |
| 3: 非線形構造 | 05 木・BST, 06 ヒープ, 07 ハッシュ |
| 4: グラフ | 08 グラフ探索, 09 最短経路・MST |
| 5: アルゴリズム | 10 ソート, 11 二分探索, 12 DP, 13 貪欲法 |
| 6: 実務応用 | 14 最適化応用, 15 シミュレーション |

## ノートブックのコーディング規則

- セルは上から順に実行できるよう書く（依存関係を上に置く）
- 演習の答えは折りたたみセル（`# 答え` コメントで区切る）の後に書く
- 実装コードはシンプルさ優先（標準ライブラリ + numpy/scipy のみ）
- 日本語コメント推奨

## 環境

- Python 3.12 (LTS)
- パッケージ管理: uv
- 起動: `uv run jupyter lab`

## ノートブックの追加・修正時の注意

- 各トピックの `learn/` ノートブックには「実務での使いどころ」セクションを必ず含める
- `practice/` の選択問題は選択肢A〜Dの4択、穴埋めはコードの `___` を埋める形式
- 難易度は初学者が理解できるレベルを基準とし、応用例も示す
